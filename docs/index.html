🏗️ Architecture

┌─────────────────┐    WiFi/HTTP     ┌─────────────────┐    HealthKit    ┌─────────────────┐
│ Samsung Galaxy  │ ◄──────────────► │ Local Sync      │ ◄─────────────► │ iPhone (iOS)    │
│ Watch (WearOS)  │                  │ Server          │                 │ Apple Health    │
└─────────────────┘                  └─────────────────┘                 └─────────────────┘
         │                                    │                                   │
         │           Bluetooth LE             │                                   │
         └────────────────────────────────────┼───────────────────────────────────┘
                     (Fallback)               │
                                              │
                                     ┌─────────────────┐
                                     │ SQLite Database │
                                     │ (Local Storage) │
                                     └─────────────────┘
  • Primary Transport: Local Wi-Fi Server (REST API)
  • Fallback Transport: Bluetooth Low Energy (BLE)
  • Data Storage: Local SQLite database
  • Sync Method: Timestamp-based stateful synchronization
  • Privacy: No cloud dependencies, all data local

📊 Health Data API

POST/api/v1/health-data

Submit health data from WearOS devices

{
  "deviceId": "galaxy-watch-001",
  "deviceName": "Samsung Galaxy Watch",
  "deviceType": "wearos",
  "dataType": "heart_rate",
  "records": [
    {
      "timestamp": 1703936400000,
      "value": 72,
      "unit": "bpm",
      "metadata": {
        "accuracy": "high",
        "source": "sensor"
      }
    }
  ]
}

GET/api/v1/health-data

Retrieve health data with filtering options

Query Parameters:

  • deviceId - Filter by device ID
  • dataType - Filter by data type
  • since - Timestamp to get data after
  • until - Timestamp to get data before
  • limit - Maximum records (default: 1000)
  • offset - Pagination offset

GET/api/v1/health-data/types

Get supported health data types and validation rules

🔄 Sync Management API

POST/api/v1/sync/register

Register a device for synchronization

{
  "deviceId": "galaxy-watch-001",
  "deviceName": "Samsung Galaxy Watch",
  "deviceType": "wearos",
  "metadata": {
    "manufacturer": "Samsung",
    "model": "Galaxy Watch 4"
  }
}

GET/api/v1/sync/status/:deviceId

Get sync status and statistics for a device

GET/api/v1/sync/data/:deviceId

Retrieve new health data for iOS app sync

Query Parameters:

  • since - Get data after this timestamp
  • dataType - Filter by specific data type

GET/api/v1/sync/devices

List all registered devices with sync statistics

📡 Bluetooth API

GET/api/v1/bluetooth/status

Get Bluetooth service status and connected devices

POST/api/v1/bluetooth/start

Start Bluetooth advertising for fallback sync

POST/api/v1/bluetooth/stop

Stop Bluetooth advertising

POST/api/v1/bluetooth/test

Test Bluetooth functionality and capabilities
